# workflow-cookbook v1.1.3

## Summary

Public-facing polish release for product readiness follow-up.
This release improves release traceability, tightens acceptance navigation,
and adds a docs review due checker for ongoing maintenance.

## Highlights

- Added release mapping to the acceptance index so `task / acceptance / release`
  relationships are easier to follow.
- Added `tools/ci/check_docs_review_due.py` to detect overdue document reviews.
- Clarified product readiness wording so published docs do not overstate
  completed rollout status.
- Removed README front matter from public-facing multilingual README files so
  GitHub renders cleaner entry pages.

## Changes

### Docs / Product Readiness

- Updated `docs/acceptance/INDEX.md` with release mapping and cleaner linkage.
- Updated `docs/addenda/R_Product_Readiness_Checklist.md` to match the actual
  implementation state.
- Updated `RUNBOOK.md` and `CHECKLISTS.md` with docs review due guidance.

### CI / Tooling

- Added `tools/ci/check_docs_review_due.py`.
- Extended `tools/ci/generate_acceptance_index_standalone.py` to emit release
  linkage.
- Updated
  `tests/test_generate_acceptance_index_standalone.py` to cover relative-path
  output.

### Public Repository View

- Removed front matter from `README.md`, `README.ja.md`, and `README.zh-CN.md`
  so repository landing pages render without internal metadata noise.
